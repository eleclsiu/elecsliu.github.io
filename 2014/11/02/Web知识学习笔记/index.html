缓存

本地缓存

  • 将不需要每次重复更新的icon图标、图片等在本地存储，加快网页响应。
  • 但有时需要定期更新，常用的定期更新机制有：设置更新标志、判断最后更新时间、定时强制更新等

服务器缓存

  • 例子：打分性的排行榜，计算量大，不能每个用户的每次请求都去计算实时的排序，这时需将结果存储起来，一段时间内都从该结果读取排行榜数据。
  • 服务器存储的主要情况：
    • 将数据库查询或计算结果缓存
    • 将多数时间内不变的页面缓存
    • 将读取频繁的数据缓存
  • 更新问题：对于更新频繁或实时性要求高的数据不适合服务器缓存。
  • 更新频次的确定要考虑计算量、使用场景、研发技术和框架等

同步异步

  • 同步：服务器更新后才返回端显示
  • 异步：用户操作立即响应，不需等服务器反馈

需要考量的问题：

  • 该操作的数据是否必须保持服务端及时更新？
  • 该操作是否有必要打断用户的使用，让用户等待？

HTTP

  • HTTP Request: 请求行、HTTP头、内容
  • HTTP Response: 状态行、HTTP头、内容

静态、动态页面

  • 静态页面：不需要服务器器端程序进行处理的页面
  • 动态页面：需要服务器端脚本进行处理之后输出的页面
  • 伪静态：可以把动态语言（比如PHP）的程序文件“.php”伪装成“.html”或者“.htm”。一方面可以避免黑客对于服务器处理逻辑的揣测，另一方面有利于SEO

程序设计模式MVP

  • 模型（Model）、视图（View）和控制器（Controller）
  • 前端工程师会侧重于V、C的开发（也就是我们看到的页面和页面的展示逻辑）
  • 后端工程师则主要侧重于M及其对应的数据库表、存储等的开发（也就是各种涉及到的对象，及对象的一些操作）
  • 如下即为研发用户注册/登录功能时的MVP示意图：
  • MVP三者相互独立，用户模型M除了可以给“注册”这个控制器使用，也可以给“登录”控制器使用。如果要更改密码验证规则，只需修改用户模型。整个代码的可维护性、开发效率大大提高了。

网页构成

HTML(HypeText Mark-up Language)

  • 网页的内容
  • 描述网页文档结构的语言，通过一系列标签定义网页内容
  • Head:描述性信息，不显示在网页
  • Body:<p>段落文字<a color="red">红色字</a></p>

CSS(Cascading Style Sheets)

  • 网页的表现
  • 精确控制网页元素展示方法，定义在Head部分
  • <style></style>

JS(Javascript)

  • 网页的行为
  • 网页中动态元素（评论展开、输入框文字变化等）的标签和属性的脚本语言
  • <script></script>

  • 对于浏览器窗口的操作
